# Installation and deployment paths

Choose how you run agent-detective on a host or in your cluster. This page is the **entry point**; the detailed guides are linked below.

:::tip[Reading order]
Use the **Configuration** section in the sidebar after you pick an install path—load order and env are documented there.
:::

**Typical reading order:** this page (choose how you run) → [configuration hub](../config/configuration-hub.md) (how settings load) → [upgrading](upgrading.md) (releases and image tags).

## Choose a deployment style

<Tabs syncKey="deploy">
  <TabItem label="Container (GHCR)" icon="seti:docker">
    **When to use:** Production on a single host or small team; minimal build steps.

    **What you need:** Docker (or compatible runtime), a `config/` directory to mount, secrets via env (see [configuration.md](../config/configuration.md)).

    ```bash title="Pull and run"
    docker pull ghcr.io/toniop99/agent-detective:latest
    docker run -p 3000:3000 -v ./config:/app/config ghcr.io/toniop99/agent-detective:latest
    ```

    See [docker.md](docker.md) for compose files and production configuration.
  </TabItem>
  <TabItem label="Docker Compose" icon="seti:docker">
    **When to use:** Same as container, but you want `compose` and optional bind mounts for `config/` and `plugins/`.

    **What you need:** Docker Compose, plus a compose file — [docker.md](docker.md) includes `docker-compose.ghcr.yml` for pull-only mode.

    ```bash title="Start with Compose"
    docker compose up -d
    ```
  </TabItem>
  <TabItem label="Native binary" icon="seti:binary">
    **When to use:** Single executable on a host with no system Node.js or pnpm; best for bare-metal installs and simple upgrades.

    **What you need:** Download the binary release asset, a `config/` directory, and the configured agent CLI(s) on `PATH` (use `doctor` to verify).

    See [binary.md](binary.md) for install layout, `--config-root`, plugins, and systemd.
  </TabItem>
  <TabItem label="From source" icon="seti:typescript">
    **When to use:** You fork the repo, change `packages/`, or run without a prebuilt image.

    **What you need:** Node.js 24+, pnpm 10+ (see root `package.json` `packageManager`), git.

    ```bash title="Clone and build"
    git clone https://github.com/toniop99/agent-detective.git
    cd agent-detective
    pnpm install
    pnpm run build
    pnpm run build:app
    pnpm start
    ```
  </TabItem>
  <TabItem label="Bare metal" icon="linux">
    **When to use:** No Docker; long-running service on a VM with nginx or similar.

    **What you need:** See [deployment.md](deployment.md) for systemd unit, nginx reverse proxy, sizing, and health checks.
  </TabItem>
</Tabs>

:::note
**Kubernetes or Helm:** this repository does not ship charts. Run the [GHCR image](docker.md#published-image-ghcr) with your platform's standard workload manifest and the same `config` + env model as in [docker.md](docker.md).
:::

## Host capabilities

- **Process:** either the container image (includes a bundled `node` + built app) or Node.js 24+ when building from source.
- **Configuration:** JSON under `config/` ([configuration.md](../config/configuration.md)); optional `config/local.json` (often gitignored) for secrets and overrides.
- **Repositories:** the **local-repos** plugin needs **git** and filesystem access to the repos you list in config (bind mounts in Docker, or local paths on bare metal).
- **Network:** outbound to Jira (if you use the Jira plugin), to git remotes (for [pr-pipeline](../config/configuration.md#pr-pipeline-agent-detectivepr-pipeline)), and to your AI provider as required by the agent CLI (e.g. OpenCode). Inbound: HTTP(S) to the app (webhooks, API).
- **Agent CLIs:** the image or host must be able to run the configured agent (e.g. `opencode` in the default image). See [docker.md](docker.md#image-targets-dockerfile) and [cursor-agent.md](../development/cursor-agent.md) for adding other agents.

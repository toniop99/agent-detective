# Maintainer: releasing binaries

This page documents how maintainers cut a new release so GitHub Actions publishes:

- Docker images (GHCR) via `.github/workflows/release.yml`
- Native binaries + checksums + SBOM + signatures via `.github/workflows/binary.yml`

## Prerequisites

- You have push permission to create git tags in the repository.
- CI on `main` is green.

## 1) Update versions

Before tagging, bump versions so the binary reports the correct version:

- Root app version: `package.json` and `src/version.ts`
- First-party plugin versions (if you want them reflected in logs): each plugin package’s `packages/*/src/index.ts` version field

## 2) Merge to `main`

Release workflows run from the tag’s commit. Ensure the commit you tag is on `main`.

## 3) Create and push a tag

Release workflows trigger on tags matching `v*.*.*`.

```bash
git checkout main
git pull
git tag vX.Y.Z
git push origin vX.Y.Z
```

## 4) Verify GitHub Actions outputs

Re-running a failed workflow in the Actions UI **only re-runs the same commit** as the original tag. If the SEA build fix landed after your tag, push a **new** semver tag on the fixed commit (or move the tag if your release policy allows force-updating tags).

After the tag is pushed:

- `release.yml` publishes multi-arch Docker images and creates the GitHub Release
- `binary.yml` (pinned **Node 25.9.x** in Actions for integrated `node --build-sea`; bump the pin in the workflow when you adopt a newer 25 release) uploads:
  - `agent-detective-linux-x64`
  - `agent-detective-linux-arm64`
  - `agent-detective-darwin-arm64`
  - `checksums-*.txt`
  - `sbom-*.cdx.json` (CycloneDX)
  - `*.sigstore.json` (Cosign v3 Sigstore bundles for the binary, checksums, and SBOM)

## 5) Smoke test (recommended)

- Download the binary asset for your platform
- Run:
  - `./agent-detective-* --help`
  - `./agent-detective-* validate-config --config-root <install-dir>`
  - `./agent-detective-* doctor --config-root <install-dir>`

## Notes

- The native binary build embeds the SEA entrypoint generated by `pnpm run build:sea:app` under `dist-sea/index.cjs`, then runs `node --build-sea` on a small `sea-config.json` in CI (requires Node **25+**; Node 24 does not ship `--build-sea`).

## Local smoke (optional)

Use Node **25 or newer** locally so it matches CI, then:

```bash
pnpm run build && pnpm run build:sea:app
mkdir -p dist
cat > sea-config.json <<'EOF'
{
  "main": "dist-sea/index.cjs",
  "output": "dist/agent-detective-smoke",
  "disableExperimentalSEAWarning": true
}
EOF
node --build-sea sea-config.json
chmod +x dist/agent-detective-smoke
./dist/agent-detective-smoke --help
```

If you must build on Node 24, use the [Node SEA guide](https://nodejs.org/api/single-executable-applications.html) (`--experimental-sea-config` + [postject](https://github.com/nodejs/postject)) instead of `--build-sea`.

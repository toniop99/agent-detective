# Maintainer: releasing binaries

This page documents how maintainers cut a new release so GitHub Actions publishes:

- Docker images (GHCR) via `.github/workflows/release.yml`
- Native binaries + checksums + SBOM + signatures via `.github/workflows/binary.yml`

## Prerequisites

- You have push permission to create git tags in the repository.
- CI on `main` is green.

## 1) Update versions

Before tagging, bump versions so the binary reports the correct version:

- Root app version: `package.json` and `src/version.ts`
- First-party plugin versions (if you want them reflected in logs): each plugin package’s `packages/*/src/index.ts` version field

## 2) Merge to `main`

Release workflows run from the tag’s commit. Ensure the commit you tag is on `main`.

## 3) Create and push a tag

Release workflows trigger on tags matching `v*.*.*`.

```bash
git checkout main
git pull
git tag vX.Y.Z
git push origin vX.Y.Z
```

## 4) Verify GitHub Actions outputs

After the tag is pushed:

- `release.yml` publishes multi-arch Docker images and creates the GitHub Release
- `binary.yml` uploads:
  - `agent-detective-linux-x64`
  - `agent-detective-linux-arm64`
  - `agent-detective-darwin-arm64`
  - `checksums-*.txt`
  - `sbom-*.cdx.json` (CycloneDX)
  - `*.sig` / `*.cert` (cosign keyless signatures + certificates)

## 5) Smoke test (recommended)

- Download the binary asset for your platform
- Run:
  - `./agent-detective-* --help`
  - `./agent-detective-* validate-config --config-root <install-dir>`
  - `./agent-detective-* doctor --config-root <install-dir>`

## Notes

- The native binary build embeds the SEA entrypoint generated by `pnpm run build:sea:app` under `dist-sea/index.cjs`.
